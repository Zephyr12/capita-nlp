Data Mining Pipelines

The purpose of our data mining pipeline is to be able to have access to social media in a useful and accessible format that is reliably and periodically captured. This focuses on capturing a variety of social media data sources from relevant social media about education, capturing data about specific organisations and people in the education domain and reliably capturing this data periodically.

We decided to update our information daily, but this process can be done more often. However, it would be counter productive and comes with unwanted risks: receiving more data than the system can process, which would result in a system failure.


The three main sources at the beginning were : Twitter, Facebook, MumsNet and other domain-specific forums. Eventually, because of the lack of important content of MumsNet, we switched to scraping The Student Room instead - which required the same scraping technology. For every individual data source, we had to consider a specific API to work with.



Facebook SDK for Python

WHY FACEBOOK API?

In order to create a Facebook pipeline, we had a few options: either using the Facebook provided SDK for Python, or use other libraries that try to make it easier for the developers to work with the API. We decided that using other libraries would make our initial developing process easy (achieving a working Facebook pipeline), but it would come with risks that we could not take such as unwanted and unknown specific bugs inside that library, as they are not meant to be used for scraping. Because our system is so complex and the Data Pipeline component is the first component of it, we thought that using the Facebook provided API is the best solution, despite it's poor documentation and deprecated functionalities in time[1].


RAISED ISSUES

As we have specified the deprecated functionalities, we came to realise that Facebook did provide support for Public Feed API (which was our main request for scraping the desired information), but they restricted the access to the Feed API back in 2015 - which left us wondering what solution is most appropriate in order to get the desired information from Facebook.

SOLUTION

Even though we could not make use of the public feed of Facebook posts, we could still make search query requests to the Facebook API. As a result, we can have access to search events that are related to any UK Educational Institution. Thus, this pipeline is set up to yield event descriptions (together with other relevant information as concerned university, location, dates).

Scraping The Student Room with SCRAPY

WHY SCRAPY?

In order to store all the posts relating the Universities in UK that are found on The Student Room forum, we had two options: Scrapy or Beautiful Soup.

While BeautifulSoup is just a library for parsing html(and xml) style documents, Scrapy is a full framework for web crawling which provides the tools to manage every stage of web crawl - tools that come with certain advantages. Some of these tools are:

  1. Request manager: provides the speed of cuncurrency without needing to invest time in developing the concurrent architecture, as it is all done behind the scenes.
  2. Selectors: which are used to parse the HTML documents in order to find the bits that are needed to be parsed. (BeautifulSoup also does this).
  3. Pipelines: once the data is retrieved, it can be passed through various pipelines, which serves the purpose of our application.

As a result, we decided that Scrapy will be not only easier to implement (which also brings the advantage of being easier to debug and maintain), but actually faster because of the way it handles cocunrrency.


RAISED ISSUES

Because of the way Scrapy works, even though it produced the desired output on it's own (yielding forum posts), it did not behave as expected when we tried to merge the pipelines. Scrapy runs more threads in order to be faster and more efficient, but in order to be able to send signals between these threads, it needs to run on the main one in order - which it didn't at the stage of merging the pipes.

SOLUTION

Looked up the Source Code for Scrapy to understand how it starts its crawler and then modified the thread invocation.
