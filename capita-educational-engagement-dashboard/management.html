API Documentation

The API uses a principle called HATEOAS, which means the API returns context sensitive links that allow the client to traverse the API without having to manually construct URLS.

An example of this is in the paginated list endpoints.




{
  "count": 20,
  "data": [...],
  "next_page": "http://localhost:5000/schools/University%20College%20London/posts/?n=20&p=2&before=0&after=1521739727&field="
}

Which contains a link on to the next page if and only if that page exists and isn't empty. 


{
  "count": 14,
  "data": [...],
}

URL
GET /schools/
Description
Get a paginated list of schools.
Query Parameters
  • n: the number of results to return
  • p: the offset into the list of results. Starts at 1.
  • field: filters the list of schools by name using the value of field as a prefix.

URL
GET /schools/<school_id>
Description
Returns a detailed view of a school with school id school_id
Query Parameters
None.

URL
GET /schools/<school_id>/posts/
Description
Returns a paginated list of posts.
Query Parameters
  • n: the number of results to return
  • p: the offset into the list of results. Starts at 1.
  • field: filters the list of posts by name using the value of field.
  • before: defines the earliest date that appears in this list.
  • after: defines the latest date that appears in this list.

URL
GET /schools/<school_id>/topics/
Description
Returns a paginated list of topics.
Query Parameters
  • n: the number of results to return
  • p: the offset into the list of results. Starts at 1.
  • field: filters the list of topics by name using the value of field.
  • before: defines the earliest date that appears in this list.
  • after: defines the latest date that appears in this list.

URL
GET /schools/<school_id>/topics/<topic_id>
Description
Returns a detailed view of the topic with topic id topic_id. That belongs to the school with school id school_id
Query Parameters
None.

URL
GET /schools/<school_id>/topics/<topic_id>/posts/
Description
Returns a paginated list of posts associated with the given topic and school.
Query Parameters
  • n: the number of results to return
  • p: the offset into the list of results. Starts at 1.
  • field: filters the list of posts by name using the value of field.
  • before: defines the earliest date that appears in this list.
  • after: defines the latest date that appears in this list.


Installation Manual

A guide to set up the application.



This installation guide assumes a fully up-to-date, working and functional Linux distribution with Python3, PostgreSQL and git installed


	$ unzip capita-nlp.zip
	$ cd capita-nlp
	$ virtualenv venv
	$ source venv/bin/activate
	$ pip install -r requirements.txt
	$ cd react_frontend
	$ npm install
	$ cd ..
	$ nano src/sources.txt # add your twitter key to this file
	$ nano src/fb.py # add your facebook api key to this file
	$ nano src/conf.py # add your database credentials and the schools you wish to track here

User Manual

A simple guide to use the application after installation.




There are four main tasks that you typically need to perform. These are: starting the data processing pipeline, generating topics, starting the backend API server and starting the frontend application server.

Preliminary steps

Before performing any of these tasks you first need to initialise your environment variables. You can do this by navigating to $VENV_ROOT/bin/ and running source activate, where $VENV_ROOT is the root directory of the virtual environment during installation.

Starting the data processing pipeline

To start the data processing pipeline, navigate to the root of the project and execute python3 src/manage.py data_pipeline.

Generating topics

To regenerate topics, navigate to the root of the project and execute python3 src/manage.py retopic.

Starting the backend API server

To start the backend API server, navigate to the root of the project and execute python3 src/api.py.

Starting the frontend application server

To start the frontend application server, navigate to $PROJECT_ROOT/react_frontend/ and run npm start.
